# /tdd コマンド

TDDワークフローを開始する。

## 使い方

```
/tdd [問題番号または機能名]
```

## 例

```
/tdd Q005
/tdd iterator
/tdd flyweight
```

## 動作

1. 対象のテストファイルを特定または作成
2. 失敗するテストを作成（Red）
3. テストを通す実装を行う（Green）
4. コードを改善する（Refactor）

## TDDサイクル

### Red（赤）
- まず失敗するテストを書く
- テストが正しく失敗することを確認

### Green（緑）
- テストを通す最小限の実装
- 完璧さより動作を優先

### Refactor（リファクタリング）
- テストが通る状態を維持しながら改善
- 重複排除、命名改善など

## テスト実行コマンド

```bash
# 全テスト
rake test

# 特定のテスト
ruby -Ilib -Itest test/path/to/test_file.rb
```
