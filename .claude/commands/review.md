# /review コマンド

コードレビューを実行する。

## 使い方

```
/review [ファイルパスまたはパターン]
```

## 例

```
/review lib/decorator/
/review lib/little_rubyist.rb
/review test/composite/composite_e2e_test.rb
```

## 動作

1. 指定されたファイルまたはディレクトリのコードを読み込む
2. code-reviewerエージェントの観点に基づいてレビューを実施
3. 良い点と改善提案を出力

## レビュー観点

- Rubyらしさ（イディオム、命名規則）
- 設計（SOLID原則、デザインパターン）
- 可読性（メソッドの長さ、責務の分離）
- テスト（カバレッジ、テストの意図）

## 注意

- 学習用コードベースのため、エラーハンドリングやバリデーションの不足は指摘しない
- パターンの本質と設計意図に焦点を当てる
