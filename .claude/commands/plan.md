# /plan コマンド

新機能やデザインパターンの実装計画を立案する。

## 使い方

```
/plan [実装したい機能やパターン]
```

## 例

```
/plan flyweightパターン
/plan Q089 行列の転置
/plan observerパターンにログ機能追加
```

## 動作

1. 要件の明確化
2. 既存コードの調査（類似パターンの参照）
3. 実装ステップの洗い出し
4. テスト戦略の決定

## 出力内容

- 概要: 実装する内容の説明
- 変更ファイル: 作成/変更するファイル一覧
- 実装ステップ: 段階的な実装手順
- テスト方針: 必要なテストケース

## 計画の方針

- シンプルさを重視（学習用途）
- パターンの本質が分かる実装
- 過度な抽象化は避ける
